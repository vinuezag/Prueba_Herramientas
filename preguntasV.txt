¿Qué es el versionamiento en el desarrollo de software?
1.1 Un proceso para gestionar cambios en el código
¿Qué es la "historia de commits"?
2.2La "historia de commits" en Git es un registro de todas las confirmaciones en un repositorio, que incluye el autor, la fecha y un mensaje de cambios. Permite a los desarrolladores revisar y colaborar en el código de manera efectiva.
¿Qué es un repositorio "clonado"?
3.1 Un repositorio "clonado" es una copia completa de un repositorio existente, incluyendo todos los archivos y el historial de cambios. Se crea usando el comando git clone.
¿Qué son las "ramas" o "branches" en el desarrollo colaborativo?
4.1 Las "ramas" o "branches" en el desarrollo colaborativo son líneas de trabajo independientes que permiten a los desarrolladores trabajar en características o correcciones sin afectar el código principal. Facilitan la colaboración al permitir realizar cambios aislados y luego fusionarlos con la rama principal una vez completados.
¿Qué es un "fork" en GitHub?
5.1 Respuesta
Un "fork" en GitHub es una copia de un repositorio que se crea en la cuenta de un usuario, permitiendo trabajar de forma independiente del original. Esto facilita realizar cambios y experimentar sin afectar el proyecto original, y luego se puede enviar un "pull request" para proponer la inclusión de esos cambios en el repositorio original.
